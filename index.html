Getting Started

DO Framework, or -DOFW- for short is an options framework and administration pages management plugin for WordPress which comes with various handy tools to make the life of a WordPress developer easier.

Quick example on using the plugin to add an example page and a couple of fields to it in 2 easy steps.

Step #1) Add an admin page:

                                    $pageInstance = dofw_admin_page([
                                        'id' => 'PAGE-ID',
                                        'parent' => null,
                                        'menu_title' => 'Example Page',
                                        'menu_side_title' => 'Example',
                                        'title' => 'Example Page',
                                        'type' => 'form',
                                    ]);

Step #2) Add fields to the page:

                                    $pageInstance->add_fields([
                                        'FIELD-ID' => [
                                            'type' => 'text',
                                            'label' => 'Example Text Input',
                                        ],
                                    ]);

As you can see it is quite easy and simple to use the plugin for adding pages and fields.

How to Embed the Plugin

First, include the autoload file

Wether you're using DOFW with another plugin or a theme, you must first include the plugin autoload.php file before proceeding with using its functionality.

                            require_once('path/to/dofw/autoload.php');

Using DOFW with another plugin or theme

Please make sure to purchase the extended license before embedding the plugin with your work to allow for distribution without copyright infringement.

You can use DOFW with other plugin(s) either by using it as a required peer plugin which can be achieved by simply adding it to WordPress plugins directory.

Or, you can include plugin files under your plugin or theme directory, here is an example of your plugin structure after adding DOFW to a plugin: 

                            /my_plugin
                                /dofw
                                ...
                                /includes
                                /assets
                                /templates
                                my_plugin.php

After adding DOFW files to your plugin make sure to require or include the plugin autoload.php at the top of your plugin front controller, in this example it is my_plugin.php.

In case you're using DOFW with a theme, make sure to require or include the plugin autoload.php at the top of your theme's functions.php.

Adding Admin Pages

Adding a new admin page:

Since there are many types of administration pages out there, DOFW supports some of these types out of the box.
To add a new form page, specify the page type parameter to form,
here is an example: 

                            $adminPage = dofw_admin_page([
                                // Page ID (Used later to access page instnace)
                                'id' => 'example_form',
                                
                                // Enable adding the page to the side-menu or hide it instead.
                                'menu_enabled' => true,

                                // Parent Menu ID (Leave null to assign to the top-level admin sidebar menu)
                                'parent' => null,

                                // WordPress Admin Menu Title
                                'menu_title' => 'Example Form',

                                // WordPress Admin Sub-Menu Title
                                'menu_side_title' => 'Example Form',

                                // Page Title
                                'title' => 'Example Form',

                                // Page Type
                                'type' => 'form',

                                // Enable / disable form page sidebar.
                                'has_sidebar' => true,

                                // Menu position
                                'position' => 2,

                                // Menu Icon
                                'icon' => DOFW_IMAGES_URL . 'dedulab.svg',

                                // Required user capability to view this page
                                'capability' => 'manage_options',

                                // Tab Menu Options
                                'tab_link' => [
                                    'label' => 'Example Form',
                                    'icon' => 'cog',
                                    'position' => 'start',
                                ],
                            ]);

There are also other types of pages available, here are a list of them:

  • dashboard: The primary options panel page dashboard template.
  • demos: A list of available demos.
  • plugins: A list of required / recommended plugins.
  • form: A form page that can be used for displaying form sections, groups and fields.
  • backup: Options import / export page.
  • activation: Activation page.

Adding Form Sections

DOFW form page fields are grouped using a set of sections and groups.
Sections may contain one or more group(s), and their links is displayed in form page sidebar.
Groups may contain one or more field(s).

Adding a new section:

Sections can be added to form pages by first getting an instance of the page, then adding a new section or multiple sections using the $pageInstance->add_section(), or $pageInstance->add_sections() methods.
Here is an example: 

                            // Get page instance
                            $pageInstance = dofw_get_page('example_form');
                            
                            // Adding a sinlge section
                            $pageInstance->add_section( 'section1_id', [
                                'icon' => 'cog',
                                'label' => 'Compact Settings',
                            ]);

                            // Adding multiple sections
                            $pageInstance->add_sections([
                                'section2_id' => [
                                    'icon' => 'cogs',
                                    'label' => 'Advanced Settings',
                                ],
                                'section3_id' => [
                                    'icon' => 'brush',
                                    'label' => 'More Settings',
                                ],
                            ]);

  Remember that sections are the root level of your form parts, they contain groups of fields, and their IDs are used later to associate groups and fields with their related section.

API Reference:

  • Function: dofw_get_page()
  • Method: Do_Framework_Admin_Page::add_section()
  • Method: Do_Framework_Admin_Page::add_sections()

Adding Form Groups

Adding a new group:

Groups can be added to form pages by first getting an instance of the page, then adding a new group or multiple groups using the $pageInstance->add_group(), or $pageInstance->add_groups() methods.
Here is an example: 

                            // Get page instance
                            $pageInstance = dofw_get_page('example_form');
                            
                            // Adding a sinlge group
                            $pageInstance->add_group( 'group1_id', [
                                'icon' => 'cog',
                                'title' => 'Compact Settings',
                                'section' => 'SECTION-ID',
                            ]);

                            // Adding multiple groups
                            $pageInstance->add_groups([
                                'group2_id' => [
                                    'icon' => 'cogs',
                                    'title' => 'Advanced Settings',
                                    'section' => 'SECTION-ID',
                                ],
                                'group3_id' => [
                                    'icon' => 'brush',
                                    'title' => 'More Settings',
                                    'section' => 'SECTION-ID',
                                ],
                            ]);

Here are all the available group configuration parameters: 

                            $pageInstance->add_group( 'group1_id', [
                                // Icon can be a font awesome icon class, or a full URL to a PNG or SVG image.
                                'icon' => 'cog',

                                // Note that a group uses the parameter "title" not "label"
                                'title' => 'Compact Settings',

                                // Specify parent section ID
                                'section' => 'SECTION-ID',

                                // Allow folding the group (Show / Hide) its fields, "true" by default
                                'fold' => true,

                                // Default group toggle state (Is hidden or displayed), "true" by default
                                'open' => true,
                            ]);

  Remember that groups are fields collections, they contain one or more fields, and their IDs are used later to associate fields with their related group.

API Reference:

  • Function: dofw_get_page()
  • Method: Do_Framework_Admin_Page::add_group()
  • Method: Do_Framework_Admin_Page::add_groups()

Adding Form Fields

Adding a new field:

Fields can be added to form pages by first getting an instance of the page, then adding a new field or multiple fields using the $pageInstance->add_field(), or $pageInstance->add_fields() methods.
Here is an example: 

                            // Get page instance
                            $pageInstance = dofw_get_page('example_form');
                            
                            // Adding a sinlge field
                            $pageInstance->add_field( 'field1_id', [
                                'type' => 'text',
                                'group' => 'GROUP-ID',
                                'section' => 'SECTION-ID',
                                'label' => 'Text Field',
                                'desc' => 'Text field description',
                            ]);

                            // Adding multiple fields
                            $pageInstance->add_fields([
                                'field2_id' => [
                                    'type' => 'textarea',
                                    'group' => 'GROUP-ID',
                                    'section' => 'SECTION-ID',
                                    'label' => 'Text Area Field',
                                    'desc' => 'Text area field description',
                                ],
                                'field3_id' => [
                                    'type' => 'slider',
                                    'group' => 'GROUP-ID',
                                    'section' => 'SECTION-ID',
                                    'label' => 'Slider Field',
                                    'desc' => 'Slider field description',
                                ],
                            ]);

  Check all available field types.

Here are all the available field configuration parameters regardless of field type: 

                            $pageInstance->add_field( 'field1_id', [
                                // Field type (Check below to see available field types)
                                'type' => '',
                                
                                // Parent Section ID
                                'section' => '__default',
                                
                                // Parent Group ID
                                'group' => '',
                                
                                // Field name (defaults to field ID)
                                'name' => '',

                                // Field default value
                                'default' => '',

                                // Field label
                                'label' => '',
                                
                                // Field description
                                'desc' => '',

                                // Field grid columns (based on a 16 columns grid) defaults to 16 or full width
                                'columns' => '',

                                // Help message
                                'help' => '',

                                // Items array (available only for select, checkbox, and choices based fields)
                                'items' => [],

                                // Field extra attributes
                                'attributes' => [],

                                // Field extra options (used for further field type configuration)
                                'options' => [],

                                // Show / hide field conditions based on other fields values
                                'show_if' => [],
                            ]);

Field Types

DO framework currently supports 23 field types out of the box with more types and features planned to be added in future releases, read further to learn how to implement and configure each field type to fit your needs.

  All fields must extend the Do_Framework_Field class, so if you intend to add your own field type keep in mind that your field type class must extend this class, otherwise it won't be accepted.

Available out of the box field types:

  • Text: Simple text field.
  • email: Email field.
  • url: URL field.
  • slug: SEO friendly text value field.
  • number: Number field.
  • password: Password field.
  • textarea: Simple text area field.
  • code: Code editor field.
  • editor: WordPress editor.
  • file: File / Image upload field.
  • switch: Toggle -true/false- field.
  • checkbox: Check-box field.
  • radio: Radio field.
  • select: Single / Multiple select field.
  • country: Single / Multiple select of world countries field.
  • image_select: Image select field.
  • align: Text alignment picker field.
  • space: Get valid values for margins, padding, or position space.
  • icon: Powerful icons picker field.
  • color: Powerful color picker field.
  • date: Date / Time picker field.
  • slider: Draggable slider field.

API Reference:

  • Class: Do_Framework_Field

  If you want to override the activation class used for sending purchase code validation requests, you can simply use the DOFW/Class/Activation filter to do so.

Here is an example: 

                            class My_Custom_Validation extends Do_Framework_Activation {
                                public function activate( $purchaseCode )
                                {
                                    $activated = $this->_attempt_activation( $purchaseCode );
                                    $status = 'inactive';

                                    if ( $activated === true ) {
                                        $status = 'active';

                                        $this->_set_code($purchaseCode);
                                    }

                                    $this->_set_status($status);

                                    return $activated;
                                }
                                
                                protected function _attempt_activation( $purchaseCode )
                                {
                                    $itemId = Do_Framework_Registry::get_store_id();
                                    $request = @file_get_contents("https://example.com/custom-activation-api.php?purchase_code={$purchaseCode}&item={$itemId}");
                                    $response = @json_decode( $request, true );

                                    if ( empty($response) ) {
                                        return false;
                                    }

                                    return $response['success'] === true;
                                }
                            }

                            // Override default activation class
                            add_filter('DOFW/Class/Activation', function () {
                                return My_Custom_Validation::class;
                            });

  If you want to disable activation feature including the activation page, and its related UI components, you can do so by simply defining a constant DOFW_ENABLE_ACTIVATION and set its value to false, but make sure to do so before including plugin's autoload.php file.

Otherwise, you can disable the activation functionality by using the DOFW/Activation/Enabled filter as illustrated in the example below: 

                            function my_prefix_disable_dofw_activation( $enabled ) {
                                return false;
                            }

                            add_filter('DOFW/Activation/Enabled', 'my_prefix_disable_dofw_activation');

Overriding Form Actions

A DOFW form have 2 actions which can be used for overriding the default logic or implementing form side-effects.
The first form action can be used to process form inputs before input values are stored in the database, and the second form action is triggered after a save action is finished and successful, if an error occures before or during saving form inputs, the second action will never be triggered.

First action: Before form save. 

                            $pageId = 'example_form';

                            // In this example we're intercepting an invalid input value and prevent the form from saving input values.
                            function my_custom_before_save_action( $fields )
                            {
                                global $pageId;
                                
                                $page = dofw_get_page( $pageId );

                                if ( isset($fields['my_custom_input']) && $fields['my_custom_input'] === 'invalid_value' ) {
                                    $page
                                        ->add_notice(
                                            'my_custom_notice',
                                            'Error! Input has invalid value.',
                                            'error'
                                        )
                                        ->redirect_to_referer();
                                }
                            }

                            add_action( "DOFW/Form/Save/Before/{$pageId}", 'my_custom_before_save_action' );

Second action: After form save. 

                            $pageId = 'example_form';

                            // In this example we're adding a side-effect to the save process, which validates a form input.
                            function my_custom_after_save_action( $args )
                            {
                                global $pageId;

                                $args = is_array($args) ? (Object) $args : $args;
                                $page = dofw_get_page( $pageId );
                                $valid = true;

                                if ( isset($args->data['my_custom_input']) && $args->data['my_custom_input'] === 'invalid_value' ) {
                                    $valid = false;
                                }

                                $page->add_notice(
                                    'my_custom_notice',
                                    $valid ? 'Input was valid.' : 'Error! Input has invalid value.',
                                    $valid ? 'success' : 'error'
                                );
                            }

                            add_action( "DOFW/Form/Save/After/{$pageId}", 'my_custom_after_save_action' );

API Reference:

  • Function: dofw_get_page()
  • Method: Do_Framework_Admin_Page::add_notice()
  • Method: Do_Framework_Admin_Page::redirect_to_referer()
